# Formatting Ownership Triage

## Problem/Feature Description

A travel app team is organizing a small backlog before starting implementation. They are not translating the app this quarter, but the next milestone includes editable follower counts like `12.5K`, trip date ranges, destination URL cards, prices, and a separate copy cleanup involving String Catalogs, plurals, and right-to-left layout review.

The maintainer wants a short scope note so the formatting work lands with the right Apple-platform skill without hiding real localization concerns.

## Output Specification

Create `formatting-ownership-routing.md` with:

- which work should be owned by the formatting skill,
- which work should be routed to the localization skill,
- the minimal locale-sensitive review that still applies before translation,
- and a final routed checklist.
